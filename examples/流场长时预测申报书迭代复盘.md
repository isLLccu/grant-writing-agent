# 案例：粤港澳大湾区近岸海表流速场长时预测申报书迭代复盘

> 项目：深圳市面上项目申报——基于扩散模型的大湾区公里级近岸海表流速场约15天集合预报方法研究  
> 迭代周期：2026年5月18日—6月9日（12个版本）  
> 团队：张广涛（PI）、林乐珊、马浩冉

---

## 申报书最终结构

```
1. 立项依据
   1.1 研究背景及意义
   1.2 国内外研究现状及发展动态分析
   1.3 已有研究的关键瓶颈和本项目的关键突破

2. 研究内容、研究目标及拟解决的关键科学问题
   2.1 研究内容（三部分递进）
   2.2 研究目标
   2.3 拟解决的关键科学问题

3. 技术路线与研究方案
```

---

## 最终确定的三部分研究内容

**(1) 基于小波残差扩散的近岸海表流速概率演化与高效递推建模**
- 回答：未来流场存在多解，如何用扩散模型逐步生成多条合理演化轨迹？如何用小波残差降低集合预报采样成本？

**(2) 复杂岸界约束下的水体连通传播与多跨度长时误差抑制**
- 在第一部分概率预测主干基础上，回答：规则网格邻近≠真实水体连通，复杂岸界如何嵌入递推约束？长时递推的误差如何通过多跨度分层预测抑制？

**(3) 集合流场预报的概率校准、极端风险量化与可信性评估**
- 在获得多成员轨迹后，回答：集合离散度如何转化为真实可信度？CRPS如何约束集合分布？如何输出极端流速概率和风险等级？

## 总结：这次写作学到的

1. **先问题，再方法**：专家审的是"这个问题值不值得解决"，不是"你的模型多复杂"
2. **承接比标题更重要**：三部分之间的衔接句比各部分标题更能说明递进关系
3. **文献必须对应具体论断**：每篇文献必须能指向一句具体的核心论断，不能做装饰
4. **语气要一致**：全文保持"针对…，本研究拟…"的申报书格式，大同行读起来更顺
5. **先手动理清逻辑，再让AI优化细节**：逻辑正确性无法委托给AI，AI的作用是细节打磨
