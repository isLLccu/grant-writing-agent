# 申报书写作方法论

> 本文档总结自一份深圳市面上项目申报书（粤港澳大湾区近岸海表流速场约15天集合预报）的完整迭代过程（2026年5月18日—6月9日，共12个版本）。

---

## 核心原则

### 1. 问题驱动，而非技术堆叠

申报书的每一段都应该回答：**这个研究要解决什么具体矛盾？**

- ❌ 错误：直接介绍模型结构、公式、算法细节
- ✓ 正确：先说"为什么现有方法在这里不够用"，再说"我们怎么针对这个不足做"

### 2. 三段式立项依据

研究背景与意义的开头固定为三段结构：

```
段1：研究对象重要性 + 为什么这个场景特别难
      ↓
段2：现有方法的贡献（先肯定）→ 现有方法的瓶颈（再说局限）
      ↓
段3：收束到核心矛盾（"由此，本项目聚焦……"）
```

**关键禁忌**：
- 不以"随着…的发展"开头
- 不在肯定之前直接说局限（先扬后抑）
- 段3必须用一句话点名核心矛盾，不能还在讲背景

### 3. 研究内容三段递进（非并列）

三部分研究内容必须是问题链条，不能是三个独立技术模块：

```
第一部分：解决预测/建模核心矛盾
    ↓ 形成主干方法
第二部分：在第一部分基础上，解决区域/几何/边界约束
    ↓ 使主干方法在真实场景下成立
第三部分：将前两部分结果转化为应用可用的评估/风险输出
```

**衔接句写法**：
- 第二部分必须以"在形成…的基础上，本研究进一步……"开头
- 第三部分必须以"在获得受…约束的…之后，本研究进一步……"开头

### 4. 研究目标与科学问题的格式

**研究目标**（2.2）
- 字数：≤130字/条
- 格式：`建立[对象]的[方法名称]。通过[核心手段]，突破[瓶颈]，形成[能力]。`
- 语气：陈述句

**关键科学问题**（2.3）
- 字数：≤200字/条
- 格式：先命名核心挑战，再列3-4个具体子问题，全部用问句
- 语气：疑问句

### 5. 技术路线的写法

每个技术部分按三个方面展开，每个方面内部：
```
问题背景（1句）→ 方法设计（2-3句）→ 预期效果（1句）
```

公式写法规范：
- 公式前：一句引入背景
- 公式后：说明各符号含义
- 公式末：一句收束效果

## 常见问题与修复

| 问题 | 表现 | 修复方法 |
|------|------|----------|
| 研究内容与技术路线重复 | 研究内容里出现网络结构、公式细节 | 研究内容只讲"做什么/为什么"，技术路线讲"怎么做" |
| 三部分研究内容并列 | 三部分标题结构相似，没有递进 | 检查第二、三部分是否有显式承接上一部分的句子 |
| 技术路线像实验报告 | 直接描述模型输入输出、数据集 | 每个方面以问题背景开头，而非"本项目采用…" |
| 公式前后裸奔 | 公式直接出现，没有引入和收束 | 加"为…，本研究定义…"和"其中，[符号]表示…，由此…" |
| 大同行看不懂 | 缩写未展开，口语化表达，文学化描述 | 检查所有缩写首次出现时是否给全称 |
